# Локальний Web Deploy — демонстрація MVP без публічного доступу

Запуск API **локально** (лише на вашому комп’ютері), без деплою на Render. Ніхто з інтернету не матиме доступу. Зручно для демонстрації роботи рішення MVP на захисті або перевірки перед публічним деплоєм.

---

## Що потрібно

- Python 3.10+ у віртуальному середовищі (`.venv`)
- Встановлені залежності: `pip install -r requirements.txt`
- Модель (опційно): `artifacts/best_model.joblib` — якщо є, API використовує ML; якщо немає, працює rule-based fallback

---

## Крок 1. Запуск API локально (один термінал)

У корені проєкту (PowerShell, Windows):

```powershell
.venv\Scripts\python -m uvicorn src.api.main:app --host 127.0.0.1 --port 8000
```

Або після активації venv:

```powershell
.venv\Scripts\Activate.ps1
python -m uvicorn src.api.main:app --host 127.0.0.1 --port 8000
```

- **127.0.0.1** — доступ лише з цього комп’ютера (немає публічного доступу).
- У консолі має з’явитися: `Uvicorn running on http://127.0.0.1:8000`.

Термінал залиште відкритим — поки він працює, API доступний локально.

---

## Крок 2. Перегляд роботи рішення (демонстрація)

### Варіант A — браузер (Swagger UI)

1. На тому ж комп’ютері відкрийте браузер.
2. Перейдіть за адресою:
   - **Документація API (Swagger):** http://127.0.0.1:8000/docs  
   - **ReDoc:** http://127.0.0.1:8000/redoc  
   - **Перевірка здоров’я:** http://127.0.0.1:8000/health  

3. На **/docs** можна:
   - викликати **GET /health** (перевірити, що модель завантажена);
   - викликати **POST /api/analyze** і вставити текст — отримати label, fake_score, verdict, ipso_techniques тощо.

Так можна по черзі показати 8 демо-текстів прямо в інтерфейсі.

### Варіант B — скрипт (8 демо-текстів одним запуском)

У **другому** терміналі (корінь проєкту):

```powershell
.venv\Scripts\python scripts/demo_api.py
```

Або (якщо venv активовано):

```powershell
python scripts/demo_api.py
```

Скрипт за замовчуванням звертається до `http://127.0.0.1:8000`: викликає `/health` і відправляє 8 демо-текстів на `/api/analyze`, виводить результати в консоль. Це і є демонстрація роботи MVP без публічного URL.

## Відмінність від Render (публічний деплой)

| | Локальний Web Deploy (цей документ) | Render (публічний) |
|---|-------------------------------------|---------------------|
| Доступ | Тільки з вашого ПК (127.0.0.1) | Публічний URL (https://…onrender.com) |
| Запуск | Команда в терміналі на вашому комп’ютері | Render запускає після push у репо |
| Демонстрація | Браузер на цьому ПК + demo_api.py | Будь-хто може відкрити URL у браузері |
| Коли використовувати | Захист у аудиторії з одним ПК, перевірка перед деплоєм | Демо по посиланню, перевірка з інших пристроїв |

Для **демонстрації перегляду роботи рішення MVP** без публічного доступу достатньо локального запуску та перегляду в браузері та/або запуску `scripts/demo_api.py`.
